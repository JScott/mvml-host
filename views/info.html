Why is MVML for me?

Many other frameworks provide one-off 3D web experiences and proprietary 3D browsers. MVML specializes in helping you build and share a network of first-person worlds.

Portable

  • Renders in your favourite browser
  • Built with mobile in mind

Accessible

  • Easy for non-technical artists to write
  • A consistent first-person experience

Robust

  • Built on Open Source web dev libraries
  • Exposes JS for advanced scripting

Getting Started

MVML isn't natively supported by browsers so it needs to be parsed into HTML and JavaScript. Our API makes that an easy step for you.

Quick

The quickest way to utilize this is to make an API request from an HTML page. We've built a helper script that can load your MVML from file or directly: 

<head>
  <script src="http://content.dev.mvml.net:6865/js/mvml-api.js"></script>
  <script>
    MVML.load_file("http://dev.mvml.net/mvml/spec.mvml");
  </script>
</head>

That's it! Just upload that to your website.

You can even use MVML.load_string to get the MVML from wherever is most convenient. You could even grab it from the body tag. Be careful of indentation though, MVML is very strict on whitespace.

Advanced

If you have access to server-side code you can also make a call from there. These examples show how I do it for this very page in NodeJS. It should be pretty similar across frameworks.

Privacy issues? You can also host the MVML parsing server yourself. Check out the Github page for more details.
